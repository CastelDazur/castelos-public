# Public disclosure and demo boundary

This repo shows how CastelOS thinks. It doesn't show how CastelOS works internally.

## What's here

Architecture docs explaining the system layers. Governance patterns for model lifecycle. Principles around local-first execution. Some code examples for hardware detection and similar utilities.

The goal is to show that this is a real system being built with engineering discipline. Not a landing page, not a pitch deck.

## What's not here

Core runtime code. Internal prompt configurations. Pack implementations. Optimization tables. Eval benchmarks with real numbers. Client-facing features.

Those are the product. Showing them publicly would remove the reason to build them in the first place.

## Where the line sits

If it explains a principle or a pattern, it's probably fine to publish. If it contains an implementation detail that someone could use to replicate the system, it stays private.

I'd rather show less and be credible than show everything and have nothing proprietary left.
